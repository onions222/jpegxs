# Bit-Exact 验证计划：C vs MATLAB JPEG-XS 编码器

## 目标
独立验证 C 编码器和 MATLAB 编码器对相同输入图像、相同配置参数下是否产生完全一致（byte-for-byte）的压缩码流。

## 方法

### 测试输入
| # | 输入文件 | 描述 |
|---|---------|------|
| A | `samples/input.ppm` (1391×2048, 8bit RGB) | 大图，真实场景 |
| B | `samples/debug_input.ppm` (小图) | 小图，快速验证 |

### 测试配置

| Test | 输入 | profile | 目标大小 (bytes) | 说明 |
|------|------|---------|-----------------|------|
| T1 | A | Main444.12 | 1103754 | 约 3.06 bpp，中等压缩率 |
| T2 | B | Main444.12 | 4096 | 小图固定大小 |

### 验证步骤

1. **清理**：删除所有 `/private/tmp/` 下的中间文件，避免残留数据干扰
2. **重新编译 C 代码**：`cmake --build` 确保使用最新源码
3. **C 编码**：对每个测试用例运行 `jxs_encoder`，生成 `.jxs` 码流
4. **MATLAB 编码**：编写独立 MATLAB 脚本，对相同输入用相同参数编码，输出 `.jxs`
5. **二进制比对**：
   - `cmp` 逐字节比对两个 `.jxs` 文件
   - `shasum -a 256` 计算两边输出的哈希，确认完全一致
6. **额外交叉验证**：用 C 解码器分别解码 C 编码和 MATLAB 编码的码流，确认解码结果也一致

### 判定标准
- ✅ 两个 `.jxs` 文件 `cmp` 无差异 且 SHA-256 一致 → **bit-exact**
- ❌ 任何字节不同 → **不是 bit-exact**，报告第一个差异位置
