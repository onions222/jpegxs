# 教材可用性与终稿前检查清单

本清单用于判断教材当前处于：

1. 不可用
2. 基本可用
3. 接近终稿

## A. 结构检查

- [ ] `main.tex` / `preamble.tex` / `README.md` 完整
- [ ] 章节主线齐全
- [ ] 算例册齐全
- [ ] 附录和符号表存在
- [ ] 教材可以成功编译

## B. 初学者检查

- [ ] 每章首次出现的关键术语都有解释
- [ ] 关键变量在首次出现附近定义
- [ ] 难章节至少有一个最小例子
- [ ] 配置表、实现映射表前有“如何读表”的说明
- [ ] 正文先给直觉，再给正式定义和公式

## C. 内容正确性检查

- [ ] 关键公式与源码一致
- [ ] 单位系统一致（byte / nibble / bit）
- [ ] fallback 路径没有被误讲成主路径
- [ ] 组级 / 块级 / packet 级粒度没有混淆
- [ ] 配置家族没有被过度概括

## D. 主链一致性检查

重点检查：

- [ ] `07 -> 08 -> 09 -> 10 -> 11 -> 12` 概念一致
- [ ] `03` 和 `07` 中重复公式一致
- [ ] `14` 的端到端追踪和主文一致
- [ ] `15` 没有残留旧心智模型
- [ ] 算例册与正文一致

## E. 高风险知识点检查

这些点只要有一个讲错，整本书就不算“说得对”：

- [ ] GCLI 是组级位深摘要，不是系数本体
- [ ] GTLI 是控制参数，不是数据统计量
- [ ] sigflag 粒度与 `S_s` 的关系讲对
- [ ] raw fallback 的位置和作用讲对
- [ ] 列宽公式讲对
- [ ] 子带总数公式讲对
- [ ] DWT 可逆性的前提讲对

## F. 算例检查

- [ ] 每本关键算例册都能独立读
- [ ] 每个关键算例都有输入、参数、中间变量、结果、代码对应
- [ ] 算例没有沿用旧说法
- [ ] 算例不是“调试痕迹”，而是正式教材表达

## G. 何时判定为“基本可用”

满足以下条件即可：

- [ ] 主线已齐
- [ ] 初学者能看懂大致链路
- [ ] 高风险知识点已修正
- [ ] 主文、算例、trace 基本同步

如果以上仍不满足，则只能算“草稿”。

## H. 何时判定为“接近终稿”

除“基本可用”外，再满足：

- [ ] `15-notes` 不再残留旧说法
- [ ] `14-end-to-end-trace` 与主文完全同步
- [ ] 关键算例册无明显旧心智模型
- [ ] 没有已知的概念性严重错误
- [ ] 剩余问题主要是排版或小措辞

## I. 剩余问题分级

### 严重

会建立错误心智模型的问题，例如：

- 把 raw fallback 说错
- 把 sigflag 粒度说错
- 关键公式错误

### 中等

主文之间不一致、notes 仍残留旧结论、算例未同步。

### 轻微

重复句、个别措辞不够顺、排版小瑕疵。

如果仍有“严重”项，教材不应标记为可用终稿。
